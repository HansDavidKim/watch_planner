# Watch Planner — Use Case Scenarios

Galaxy Watch 6용 Planner 앱의 Use Case Scenario 문서입니다.

## Actors

| Actor | 설명 |
|-------|------|
| **User** | Galaxy Watch 6을 착용한 사용자 |
| **Phone** | 연동된 Android 스마트폰 |
| **System** | Watch Planner 앱 |

---

## UC-1: Daily Routine 추가

| 항목 | 내용 |
|------|------|
| **Actor** | User |
| **목적** | 매일 반복되는 루틴(기상, 운동, 복약 등)을 등록 |
| **선행 조건** | 앱이 실행된 상태 |

### 기본 흐름 (Main Flow)
1. User가 루틴 추가 화면으로 진입한다.
2. 루틴 이름을 입력한다.
3. 반복 시간을 설정한다.
4. 반복 요일을 선택한다 (매일 / 특정 요일).
5. 알림 방식을 선택한다 (진동, 소리, 무음).
6. System이 루틴을 저장하고 메인 화면에 표시한다.

### 대안 흐름 (Alternative Flow)
- **3a.** 종일 루틴인 경우 → 시간 설정을 건너뛴다.
- **4a.** 평일만 선택 → 월~금 자동 선택.

### 후행 조건
- 설정된 시간에 알림이 트리거된다.
- 루틴 완료 시 체크 표시할 수 있다.

## UC-2: Non-daily Item 추가 (Meeting 등)

| 항목 | 내용 |
|------|------|
| **Actor** | User |
| **목적** | 회의, 약속 등 일회성/비정기 일정을 등록 |
| **선행 조건** | 앱이 실행된 상태 |

### 기본 흐름 (Main Flow)
1. User가 일정 추가 화면으로 진입한다.
2. 일정 제목을 입력한다.
3. 날짜와 시간을 설정한다.
4. (선택) 장소 또는 메모를 입력한다.
5. 사전 알림 시간을 설정한다 (5분 전, 10분 전, 30분 전 등).
6. System이 일정을 저장하고 해당 날짜에 표시한다.

### 대안 흐름 (Alternative Flow)
- **3a.** 종일 일정인 경우 → 시간 설정을 건너뛴다.
- **5a.** 알림이 필요 없는 경우 → 알림 없음 선택.

### 후행 조건
- 메인 화면의 타임라인에 일정이 표시된다.
- 설정된 사전 알림 시간에 알림이 트리거된다.

## UC-3: Alarm 설정

| 항목 | 내용 |
|------|------|
| **Actor** | User |
| **목적** | 루틴/일정에 대한 알림 방식을 세밀하게 제어 |
| **선행 조건** | 루틴 또는 일정이 최소 1개 등록된 상태 |

### 기본 흐름 (Main Flow)
1. User가 특정 루틴/일정의 알림 설정으로 진입한다.
2. 알림 유형을 선택한다:
   - **Vibration**: 진동 패턴 선택 (짧게, 길게, 반복)
   - **Sound**: 알림음 선택
   - **Silent**: 화면 표시만
3. 알림 강도를 조절한다.
4. System이 설정을 저장한다.

### 대안 흐름 (Alternative Flow)
- **2a.** Vibration + Sound 조합 선택 가능.
- **1a.** 기본 알림 설정에서 전체 기본값을 변경할 수 있다.

### 후행 조건
- 해당 루틴/일정의 알림이 설정된 방식으로 동작한다.

## UC-4: 휴대폰 연동

| 항목 | 내용 |
|------|------|
| **Actor** | User, Phone |
| **목적** | 워치와 폰 간 일정 데이터를 동기화 |
| **선행 조건** | 폰에 Companion 앱이 설치되어 있고, Bluetooth 연결 상태 |

### 기본 흐름 (Main Flow)
1. User가 워치 앱에서 연동 설정으로 진입한다.
2. 폰의 Companion 앱과 페어링한다.
3. 워치에서 추가한 루틴/일정이 폰에 동기화된다.
4. 폰에서 추가한 일정이 워치에 동기화된다.

### 대안 흐름 (Alternative Flow)
- **2a.** Bluetooth 연결 실패 시 → 재연결 안내 메시지를 표시한다.
- **3a.** 오프라인 상태 → 로컬에 저장 후, 연결 복구 시 자동 동기화.
- **4a.** 충돌 발생 시 → 최신 수정 시간 기준으로 병합하고 User에게 알린다.

### 후행 조건
- 워치와 폰의 데이터가 일치한다.
- 한쪽에서의 변경이 다른 쪽에 반영된다.
